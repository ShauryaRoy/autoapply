## 🧠 Deep Features of Career-Ops

### 1. The "A-G" Evaluation Framework
Every job URL or description passed to Career-Ops triggers a 7-block analysis:
- **Block A (Summary):** Classifies the role into technical archetypes (LLMOps, Agentic, FDE, AI PM).
- **Block B (CV Match):** Performs a line-by-line mapping from the Job Description to the candidate's `cv.md`, identifying exactly where requirements are met or missed.
- **Block C (Leveling):** Strategies for "Selling Senior without lying" and handling "Down-leveling" during negotiations.
- **Block D (Comp Research):** Uses WebSearch to pull real-time salary data from Glassdoor, Blind, and Levels.fyi.
- **Block E (Personalization):** Suggests 5 specific changes to the candidate's CV and LinkedIn to maximize match.
- **Block F (STAR+R Stories):** Generates "STAR" stories for interviews, adding a **"Reflection"** section to show seniority/wisdom.
- **Block G (Legitimacy):** **Ghost Job Detection.** Analyzes posting age, layoffs at the company, and description quality to warn the user if a posting is a "fake" opening used for pipeline building.

### 2. Tailored PDF CV Engine
Career-Ops generates a unique PDF for *every* application using a sophisticated pipeline:
- **ATS Optimization:** Specifically designed for a "6-second recruiter scan" with single-column layouts.
- **Keyword Injection:** Naturally weaves 15-20 keywords from the JD into existing bullet points without lying.
- **Visual Design:** Uses **Space Grotesk** and **DM Sans** fonts with HSL gradient headers for a premium, non-generic look.
- **Canva Integration:** A specialized mode that can duplicate a Canva design via API, replace text, and automatically re-calculate element spacing to prevent overlapping.

### 3. Automated Portal Scanner
The "Scan" mode is significantly more advanced than a typical scraper:
- **Playwright Navigation:** Directly visits "Careers" pages of 45+ top AI companies (Anthropic, OpenAI, etc.).
- **Greenhouse/Ashby APIs:** Calls the underlying ATS APIs for structured, real-time data.
- **Verification Loop:** Every link found via Google/Search is verified through a "Liveness" check to ensure it's not a dead link before adding it to your pipeline.

### 4. Interview Story Bank
As the user applies to more jobs, the system accumulates a `story-bank.md`. It identifies patterns in the user's career and maintains 5-10 "Master Stories" that are dynamically tweaked for each new interview.

## 🛠️ Comparison: Why They Are Different

### 1. Automation vs. Intelligence
*   **AutoApply** asks: *"How can I click this button and fill this text box automatically?"*
*   **Career-Ops** asks: *"Is this job worth 10 hours of my time, and how do I change my narrative to beat 500 other candidates?"*

### 2. Data Persistence
*   **AutoApply** uses a Database (PostgreSQL/Redis) suitable for a SaaS.
*   **Career-Ops** uses Markdown files and TSV (Tab-Separated Values) logs. This makes the data human-readable and allows an AI agent (like Claude) to edit the "database" directly with text.

### 3. Deployment
*   **AutoApply** requires a desktop installation and a backend server.
*   **Career-Ops** is a set of "Skills" added to a CLI coding agent (Claude Code).
